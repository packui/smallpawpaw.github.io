原因就在于多线程环境下，可能某个线程进入getInstance方法后进行判断，发现当前还没有创建实例，于是它就开始创建实例，但可能就在它还没创建成功之前，又有一个线程进入判断，发现还没有创建实例，于是它也会开始创建实例，这就会导致存在多条线程都进行了不同的实例化，产生了多个实例对象，所以可以考虑对这个部分使用同步处理，来确保单例化的正确实现

【改进方案一：声明synchroinzed关键字，对getInstance进行同步】

优点：可以保证正确的单例创建
缺点：下一个想要获取实例对象的线程必须等待上一个线程释放对象锁，效率比较低

【改进方案二：使用同步代码块】

优点：可以保证正确的单例创建
缺点：同步代码块的范围太大了，对全部代码进行了同步，和改进方案一的同步方法其实是一样的，效率比较低

【改进方案三：使用DCL双检查锁机制（Double Check Locking）】

优点：既可以保证正常的创建单例，而且因为只对关键代码进行了同步，又可以保证其它非关键代码进行异步执行，可以提高程序的效率

多线程 - 进程与线程

2018-09-05
多线程

进程与线程概念

进程： 进程是系统进行资源分配和调度的基本单位，操作系统在分配资源时是分配给进程的，一个进程至少包含一个线程，进程中的多个线程可以共享进程资源，通常我们启动一个应用程序就启动了进程，可以看作是应用级别的概念。
例如：我们启动电脑管家应用程序就会在后台启动一个进程。

线程： 线程本身不能独立存在，它是进程中的一个执行路径，也就是进程中的一个实体，操作系统分配资源时有个特殊的资源CPU是分配给线程的，因为真正占用CPU的是线程，线程之间在轮流占用CPU，所以，线程又是CPU分配的基本单位，可以看作是功能级别的概念。
例如：我们使用电脑管家的垃圾清理功能，同时又使用电脑管家的杀毒功能，那么其实就相应的启动了垃圾清理的线程、杀毒的线程。

进程与线程的内存结构

堆： 进程中最大的一块内存，在进程创建时分配的区域，存放着new构建的对象实例，可以被进程中所有线程共享。

方法区： 存放着Java虚拟机加载的类、常量、 静态变量等信息，也可以被进程中所有线程共享。

程序计数器： 即PC计数器，是线程中的一小块内存区域，用来记录线程当前要执行的指令的地址，是线程私有的，因为线程是CPU分配的基本单位，线程之间在轮流占用CPU，每当本次CPU占用的时间片用完之后，当前线程就要让出CPU的执行权，那么就在这个时候，程序计数器就要记录让出CPU时当前线程执行到哪了，等到下一次线程被重新分配到CPU时，接着这个地方继续向下执行。
备注：如果执行的是native方法，那么程序计数器记录的是undefined的地址；如果执行的是java代码，程序计数器记录的便是要执行的指令地址。

栈： 每个线程都有自己的栈区域，它存放着当前线程的局部变量、线程的调用栈帧，也是线程私有的。

启动线程的正确打开方式是调用start方法，而且调用start方法，线程并不会立即启动，这只是一个启动的标志，就像在告诉系统我已经万事俱备了，就差你给我CPU的执行机会！我就可以真正启动了！

imgcode-1.0.0使用指南

简要介绍

imgcode是一个完全由用户自定义的、灵活的、用于快速生成网页图片验证码的一个框架，图片验证码的显示效果完全交由程序员自定义。

imgcode的使用也非常简单，使用者只需提供一个Properties配置文件，通过简单设置该文件中的属性，调用方法时传入该配置对象，即可开发出属于自己的图片验证码。

Imgcode 类的方法及说明 参数说明 返回值说明
public static Map<String, BufferedImage> createImageCode(Properties properties) properties：验证码配置对象 Map<String, BufferedImage>：键表示验证码字符内容，值表示验证码图片

使用指南

1、在工程中导入 imgcode-1.0.0.jar（下载地址：https://github.com/smallpawpaw/JavaIdentifying-code/raw/master/imgcode-1.0.0.jar）
2、创建一个Properties配置文件并添加以下所有属性
imgcode-1.0.0 的所有属性介绍
属性 属性值 描述说明
width 整数值 验证码图片的长
height 整数值 验证码图片的高
bgColor RGB整数值 例如：255,255,255
borderColor RGB整数值 useBorder为true才有效
useBorder true，空值 true表示有边框，空值表示无边框
type number，letter，blend 验证码类型，number表示数字验证码，letter表示字母验证码，blend表示混合验证码
length 整数值 验证码字符长度
fontSize 整数值 验证码字符大小
fontFamily 字体值 验证码所用字体
fontColor RGB整数值，空值 空值表示颜色随机
startSpace 整数值 验证码第一个字符出现的水平位置
space 整数值 验证码字符之间的间距
angle 整数值 字符的倾斜角度的绝对值的最大值，例如：angle=20表示倾斜角度范围在 -20 到 20 之间
interfereType line，dot，空值 干扰类型，line表示线条干扰，dot表示点阵干扰，空值表示无干扰
interfereColor RGB整数值，空值 干扰色，空值表示颜色随机
interfereNum 整数值 干扰数量
interfereRadius 整数值 干扰点的大小，当interfereType=dot时才有效
